# Observability Readiness Checklist

## Entry Criteria
- [ ] System boundaries and critical user journeys identified.
- [ ] Existing logging/metrics/tracing capabilities inventoried.
- [ ] Incident history or pain points captured.
- [ ] Compliance and retention requirements clarified.

## Coverage Assessment
- [ ] Metrics gaps documented for each service/user journey.
- [ ] Logging standards defined (structure, PII handling, redaction).
- [ ] Trace instrumentation requirements outlined for key flows.
- [ ] Dashboard/reporting needs aligned with stakeholders.

## Alert Strategy
- [ ] SLOs/SLIs drafted with thresholds and burn-rate considerations.
- [ ] Alert routing and escalation paths confirmed.
- [ ] Runbooks linked or flagged for creation.
- [ ] Noise reduction tactics proposed (deduplication, suppression, maintenance windows).

## Exit Criteria
- [ ] Observability plan, backlog, and dashboard spec saved to workspace.
- [ ] Implementation dependencies and sequencing communicated.
- [ ] Follow-up validation (post-instrumentation review) scheduled or delegated.
